## 1. 项目概述

### 1.1 项目背景

随着社会经济发展和生活节奏加快，家政服务已成为现代家庭的刚性需求。传统的家政服务管理模式存在以下痛点：
- 客户信息分散，难以统一管理
- 服务订单流转效率低，派单依赖人工
- 服务人员管理混乱，缺乏标准化
- 财务结算复杂，对账困难
- 缺乏数据支撑，决策依据不足

本系统旨在构建一套完整的数字化家政服务管理平台，实现从客户获取、服务预约、订单派工、服务执行到财务结算的全流程数字化管理。

### 1.2 系统定位

- **目标用户**：家政服务公司、家政门店、保洁公司、家电清洗维修服务商
- **部署方式**：私有化部署（单租户模式，v2.0支持多租户）
- **终端覆盖**：PC管理后台 + 微信小程序（用户端）+ 服务人员H5

### 1.3 版本规划

| 版本 | 功能范围 | 计划时间 |
|------|----------|----------|
| v1.0 | 核心功能（本需求） | 12周 |
| v1.5 | 营销工具完善、服务人员APP | +4周 |
| v2.0 | 秒杀团购、智能派单、多租户 | +6周 |

## 2. 问题陈述

### 2.1 核心问题

家政服务企业在日常运营中面临以下核心问题：

1. **客户管理混乱**：客户信息分散在Excel、纸质档案、微信聊天记录中，难以统一管理和精准营销
2. **订单流转低效**：从客户下单到服务完成的流程依赖人工电话沟通，效率低且易出错
3. **服务人员调度难**：缺乏智能派单系统，服务人员排班、派工依赖经验判断
4. **财务管理复杂**：套餐卡、次卡、单次服务等多种计费方式，财务对账困难
5. **服务质量难监控**：缺乏服务过程追踪和评价机制，服务质量难以保证

### 2.2 问题影响

| 问题 | 业务影响 | 量化指标 |
|------|----------|----------|
| 客户管理混乱 | 客户流失率高，复购率低 | 客户年流失率>30% |
| 订单流转低效 | 人力成本高，客户等待时间长 | 平均派单时间>2小时 |
| 服务人员调度难 | 服务资源利用率低 | 人员闲置率>20% |
| 财务管理复杂 | 财务漏洞，对账困难 | 月度对账差异>5% |
| 服务质量难监控 | 客户投诉率高 | 月度投诉率>3% |

## 3. 目标与成功指标

### 3.1 业务目标

1. **提升客户管理效率**：实现客户信息数字化管理，支持精准营销
2. **优化订单流转效率**：实现从下单到派单的自动化，缩短响应时间
3. **提高服务人员利用率**：通过智能派单提升人员调度效率
4. **简化财务管理**：实现套餐卡、次卡、单次服务的统一计费管理
5. **保障服务质量**：建立服务过程追踪和评价机制

### 3.2 成功指标

| 指标类别 | 指标名称 | 目标值 | 测量方法 |
|----------|----------|--------|----------|
| **采用率** | 系统日活跃用户数 | >80%的在职员工 | 系统埋点统计 |
| **效率提升** | 平均派单时间 | <30分钟 | 订单创建到派单时间差 |
| **客户满意度** | NPS评分 | >50 | 月度客户调研 |
| **财务准确性** | 月度对账差异率 | <1% | 财务对账报表 |
| **服务质量** | 客户投诉率 | <1% | 客服工单统计 |
| **复购率** | 客户年复购率 | >60% | 客户消费记录分析 |

## 5. 用户画像

### 5.1 系统管理员

- **角色**：家政公司老板/系统管理员
- **目标**：全面掌控公司运营状况，做出数据驱动的决策
- **痛点**：缺乏全局视角，难以获取实时经营数据
- **使用场景**：每日查看经营报表、管理员工权限、配置系统参数

### 5.2 门店店长

- **角色**：门店负责人
- **目标**：管理门店日常运营，提升门店业绩
- **痛点**：客户跟进不及时，订单流转效率低
- **使用场景**：处理待办事项、派单调度、客户跟进、业绩统计

### 5.3 客服人员

- **角色**：客服专员
- **目标**：高效处理客户咨询和订单
- **痛点**：客户信息查询慢，订单处理流程繁琐
- **使用场景**：代客下单、处理咨询、跟进订单状态、处理投诉

### 5.4 服务人员

- **角色**：保洁阿姨/家电维修师傅
- **目标**：获取订单、完成服务、获得收入
- **痛点**：订单信息不清晰，路线规划不合理
- **使用场景**：接收派单、查看订单详情、确认服务完成

### 5.5 终端客户

- **角色**：家政服务消费者
- **目标**：便捷预约服务、享受优质服务
- **痛点**：预约流程复杂、服务质量不稳定
- **使用场景**：浏览服务、下单预约、查看订单、评价服务

### 7.4 服务工单（P0）

#### 7.4.1 工单状态流转（简化版）

```
待确认 → 待派单 → 待服务 → 服务中 → 已完成
   ↓        ↓        ↓        ↓
 已取消   已取消   已取消   已取消
```

**工单状态说明**：

| 状态 | 说明 |
|------|------|
| 待确认 | 新创建的工单，待客服确认 |

```sql
CREATE TABLE `customer` (
  `id` bigint NOT NULL AUTO_INCREMENT COMMENT '主键ID',
  `customer_no` varchar(32) NOT NULL COMMENT '客户编号',
```
