## 产品概述

养老服务预约数据抓取与分析系统，用于从全国养老服务信息平台抓取预约数据，进行深度分析和可视化展示。

## 核心功能

1. **数据抓取**：通过API接口抓取预约数据，支持未处理（ifHandle=0）和已处理（ifHandle=1）两种状态分两次获取
2. **数据处理**：数据清洗、验证、格式化，正确提取手机号（personPhone字段）、身份证脱敏
3. **可视化分析**：使用ECharts库展示处理状态、服务类型、时间趋势等多维度图表
4. **地址智能分析**：增强地址解析算法，提取省份、城市、区县、街道等多层级信息
5. **单页面应用**：数据抓取和可视化在同一页面完成，无需跳转
6. **数据导出**：支持CSV格式数据导出

## 技术栈选择

- **前端框架**：Bootstrap 5 + ECharts 5 + Vanilla JavaScript
- **后端框架**：Flask (Python)
- **数据处理**：pandas + 增强型正则表达式
- **数据存储**：CSV文件 + 内存缓存
- **部署方式**：本地Web服务（端口11261）

## 关键技术修复点

### 1. 修复ifHandle参数逻辑错误

**当前错误代码**：

```python
# app_v2.py 第122行 - 状态判断逻辑完全相反

# app_v2.py 第273-275行 - 参数调用错误

**修复方案**：

- ifHandle=0 → 获取未处理数据，状态标记为"未处理"
- ifHandle=1 → 获取已处理数据，状态标记为"已处理"

def analyze_address_enhanced(address):
    """增强型地址解析"""
    if not address:
        return {'province': '', 'city': '', 'district': '', 'street': '', 'detail': ''}
    
    # 多层正则表达式匹配
    patterns = {
        'province': r'(北京市|天津市|上海市|重庆市|河北省|山西省|辽宁省|吉林省|黑龙江省|江苏省|浙江省|安徽省|福建省|江西省|山东省|河南省|湖北省|湖南省|广东省|海南省|四川省|贵州省|云南省|陕西省|甘肃省|青海省|台湾省|内蒙古自治区|广西壮族自治区|西藏自治区|宁夏回族自治区|新疆维吾尔自治区|香港特别行政区|澳门特别行政区)',
        'city': r'(\w+市|\w+地区|\w+自治州)',
        'district': r'(\w+区|\w+县|\w+市|\w+旗|\w+自治县)',
        'street': r'(\w+街道|\w+镇|\w+乡|\w+路|\w+街|\w+大道)'
    }
    
    result = {'province': '', 'city': '', 'district': '', 'street': '', 'detail': address}
    
    # 逐层提取
    for level, pattern in patterns.items():
        match = re.search(pattern, address)
        if match:
            result[level] = match.group(1)
            # 移除已匹配的部分
            address = address.replace(result[level], '', 1)
    
    result['detail'] = address.strip()
    return result
```

## 实现注意事项

### 性能优化

- **分页加载**：API请求采用分页机制，避免单次请求数据过大
- **数据缓存**：使用内存缓存已分析的数据，减少重复计算
- **懒加载**：图表按需加载，提高页面响应速度

### 错误处理

- **API错误重试**：网络请求失败时自动重试3次
- **数据验证**：对抓取的数据进行完整性验证
- **优雅降级**：当高级功能不可用时提供基础功能

### 安全性

- **数据脱敏**：身份证号、手机号等敏感信息脱敏显示
- **输入验证**：对用户输入的curl命令进行安全验证
- **文件权限**：确保数据文件权限设置正确

### 可维护性

- **模块化设计**：各功能模块独立，便于维护和扩展
- **配置管理**：API配置、图表配置等集中管理
